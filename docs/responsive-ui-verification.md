# 响应式布局与移动端交互验收报告

## 测试分辨率

本次未启动开发服务器，未进行浏览器截图验收；完成静态代码检查、关键共享布局修复和构建验证。需要人工在以下分辨率复测：

- 375×667
- 390×844
- 430×932
- 768×1024
- 1366×768
- 1600×900
- 1920×1080

## 前台问题

- 前台主要页面已有响应式栅格和内部横向滚动。
- `/order-query` 新增页面发现长订单号、长 SKU 在窄屏可能撑宽，已补充安全换行和移动端纵向排列。
- 未发现购物车入口新增。

## 商品详情与 SKU 结果

- 本次未改商品详情业务逻辑。
- SKU 按钮和立即购买逻辑保持原代码，不引入购物车。
- 仍需人工在手机端验证多 SKU 选择、售罄状态和固定按钮遮挡。

## 立即购买与收银台结果

- 本次未改支付和订单核心链路。
- 支付会话、支付状态和未配置 Provider 降级逻辑未被响应式改动影响。
- 仍需人工验证二维码、长钱包地址、倒计时和支付方式在小屏下的实际显示。

## 用户中心结果

- `/account/orders` 已使用内部表格横向滚动和抽屉详情。
- 订单号复制、交付内容遮挡和复制逻辑保持不变。
- 需要人工复测手机端订单详情抽屉、交付内容长文本和分页控件。

## 后台导航结果

- 修复前：`AdminSidebar` 在所有宽度固定显示，手机和平板会挤压主内容；`AdminTopBar` 只有 `md:hidden` 触发器，平板缺少收起模式。
- 修复后：`AdminSidebar` 仅在 `lg` 以上固定显示，小于 `lg` 使用顶部抽屉导航。
- 移动抽屉菜单已补齐支付管理、充值管理、退款、媒体、异常中心、数据库状态、操作日志等入口。
- 抽屉导航区域设置内部滚动，菜单较多时不撑高页面。

## 后台表格结果

- 商品、订单、支付、用户、库存等宽表格继续使用内部横向滚动。
- 本次未改表格业务列和保存逻辑。
- 仍需人工在 1366×768、1600×900、1920×1080 下验证操作列、分页和横向滚动条位置。

## 分类三栏结果

- 分类管理当前在 `xl` 使用三栏，小于 `xl` 自动变为单列堆叠。
- 三栏卡片均为内部滚动容器。
- 本次未新增分步 Tabs，保留当前低风险堆叠方案。

## 弹窗与表单结果

- 分类和商品弹窗已有 `max-h-[calc(100vh-48px)]`、内容区内部滚动、标题和操作区固定。
- 小屏仍需人工验证软键盘弹出后输入框和保存按钮是否可见。

## 可访问性结果

- 后台移动导航按钮新增 `aria-label="打开后台导航"`。
- 订单查询、订单详情和后台抽屉已有关闭按钮和基础 aria 文案。
- 仍需人工键盘巡检焦点返回、表单错误关联和屏幕阅读器表现。

## 发现的问题

- 后台固定侧栏在小屏和平板下会挤压主内容。
- 移动端后台抽屉菜单不完整。
- 订单查询结果里的长订单号和商品行在窄屏可能产生横向溢出。

## 已修复的问题

- 后台固定侧栏改为 `lg` 以上显示。
- 后台移动抽屉菜单补齐并支持内部滚动。
- 订单查询页移除构建风险较高的 `useSearchParams` 依赖。
- 订单查询结果长订单号和商品行改为窄屏安全布局。

## 仍存在的问题

- 未启动开发服务器，未完成真实浏览器截图验收。
- 手机端收银台二维码、钱包地址、后台复杂表格仍需人工复测。
- 平板分类三栏目前采用单列堆叠，不是专门 Tabs 交互。
