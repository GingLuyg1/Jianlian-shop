# Jianlian Shop 商品发现、搜索与筛选验收

## 现有商品发现入口

已检查入口：

- 首页分类入口：`/`
- 首页热门入口：`/`
- 一级分类页：
  - `/products/sim-cards`
  - `/products/gift-cards`
  - `/products/digital-accounts`
  - `/products/ai-membership`
  - `/products/sms-code`
  - `/products/account-recharge`
- 商品列表：各分类页中的右侧商品列表
- 商品搜索：分类页搜索框
- 商品详情：`/products/[id]`
- 推荐商品：商品详情页推荐商品模块

发现并修复：

- 首页热门入口中存在两个硬编码直达 `/checkout?product=...` 的商品入口，已改为进入 `/products/ai-membership?search=...`，避免绕过商品详情、SKU 和可售状态校验。
- 分类页原先在浏览器端加载分类商品后再做本地搜索，已改为通过 `/api/catalog/products` 进行受控分页、筛选和排序。

未发现本次新增购物车入口。

## 商品搜索结果

新增公开接口：

```text
GET /api/catalog/products
```

搜索支持：

- 商品名称
- 商品简短说明
- 商品标识 slug
- SKU 编码
- SKU 标题
- 分类路径

处理规则：

- 搜索词会 trim 并压缩多余空格。
- 空关键词不作为有效搜索条件。
- 只返回 `active` 和 `sold_out` 商品。
- `inactive`、`draft` 商品不会出现在搜索结果。
- 停用 SKU 不作为 SKU 搜索和可售库存来源。
- 查询失败返回中文错误，前台显示重试入口，不白屏。

## 分类导航结果

分类页保留现有一级分类路由和左侧二级分类列表：

```text
一级分类
  二级分类
    商品
```

规则：

- 分类来自 `public.categories`。
- 只展示启用分类。
- 按 `sort_order` 排序。
- URL 使用 `?category=<slug>` 保存当前二级分类。
- 页面刷新后会恢复当前分类。
- 当前分类无商品时显示空状态和清空筛选入口。

本次没有新增三级分类导航。

## 筛选结果

分类页新增筛选：

- 最低价
- 最高价
- 库存状态：
  - 有库存
  - 低库存
  - 已售罄
- 交付方式
- SKU 类型：
  - 多 SKU
  - 单规格

规则：

- 筛选参数写入 URL。
- 无效价格和无效枚举会被安全忽略。
- 多 SKU 商品使用可售 SKU 汇总库存与最低价进行筛选。
- 单规格商品继续使用 `products.price` 和 `products.stock`。
- 清空筛选会恢复默认商品列表。

## 排序结果

支持排序：

- 综合排序
- 最新上架
- 价格从低到高
- 价格从高到低
- 销量优先

规则：

- 排序参数白名单校验。
- 综合排序使用售罄后置、`sort_order`、`created_at`。
- 多 SKU 商品价格排序使用最低可售 SKU 价格。
- 销量优先尝试统计已支付订单的 `order_items.quantity`；如果订单关联查询不可用，会安全降级为 0，不影响页面。

## 商品卡片结果

商品卡片显示：

- 商品主图
- 商品名称
- 简短说明
- 当前价格
- 原价
- 库存状态
- 交付方式
- 多 SKU 标识

处理规则：

- 图片失败显示品牌占位图。
- 商品名称和说明使用行数截断。
- 多 SKU 商品显示最低价或价格区间。
- 售罄商品显示“已售罄”。
- 不显示加入购物车按钮。
- 主要入口为“查看详情”与“立即购买”。

## 推荐商品结果

商品详情页新增推荐商品区域：

- 优先推荐同分类商品。
- 排除当前商品。
- 只展示公开可见商品。
- 推荐失败只显示“推荐商品加载失败”，不影响商品详情和购买按钮。
- 无推荐时显示紧凑空状态。

## 分页结果

分类页分页规则：

- 默认每页 20 条。
- 支持 20、40、60 条。
- 页码写入 URL。
- 切换搜索、筛选、排序后回到第一页。
- 上一页/下一页状态正确。
- 无数据时不显示无意义分页。

## 移动端结果

- 左侧分类和右侧筛选区域在小屏仍保持纵向布局。
- 筛选控件允许换行。
- 商品列表仍为内部滚动，不新增购物车功能。

## 发现的问题

1. 首页热门入口存在硬编码 checkout 链接。
2. 分类页搜索和筛选主要在前端对已加载结果处理。
3. 商品详情页缺少相关推荐降级模块。

## 已修复的问题

1. 首页硬编码 checkout 入口已改为分类搜索入口。
2. 新增 `/api/catalog/products`，前台通过受控 API 分页查询。
3. 分类页支持 URL 可恢复的搜索、价格、库存、交付方式、多 SKU、排序和分页。
4. 商品详情页新增推荐商品模块。
5. 通用商品卡片清理乱码文案并移除购物车语义。

## 仍存在的问题

- 销量统计依赖 `order_items` 与 `orders` 的 PostgREST 关系名；若生产库关系缓存未刷新或关系名不同，会安全降级为 0，需要后续在数据库侧增加专用聚合 RPC。
- 当前商品发现 API 为服务端受控筛选，内部最多读取 1000 条候选商品。商品规模继续增大后，建议迁移为数据库 RPC 或物化视图。

## 需要执行的 migration

本次无新增 migration。
