# 退款操作 Runbook

## 适用范围

本 Runbook 用于 Jianlian Shop 退款申请、管理员审核、余额退款、外部渠道人工退款登记和数字商品退款限制处理。

## 操作前检查

1. 确认管理员为超级管理员。
2. 打开 `/admin/refunds`。
3. 搜索退款单号、订单号或用户邮箱。
4. 打开退款详情，核对：
   - 订单号
   - 用户邮箱
   - 支付方式
   - 申请金额
   - 当前可退款金额
   - 交付状态
   - 历史退款
5. 如果是数字商品且已交付，必须额外确认业务退款政策。

## 余额支付退款

适用条件：

```text
orders.payment_method = balance
refund_requests.refund_method = balance
```

操作：

1. 选择“批准余额退款”。
2. 填写批准金额。
3. 填写管理员审核备注。
4. 二次确认。
5. 提交后系统会在数据库事务中：
   - 增加用户余额
   - 创建余额退款流水
   - 更新退款状态
   - 更新订单退款金额
   - 写入订单事件和审计日志

禁止事项：

- 不允许手工直接修改用户余额字段。
- 不允许外部支付订单走余额退款自动完成。

## 外部渠道退款

适用渠道：支付宝、微信支付、币安、USDT-TRC20、USDT-BEP20。

当前状态：未接入自动退款 Provider。

操作：

1. 选择“标记处理中”，说明正在人工处理。
2. 在线下或对应渠道完成真实退款。
3. 回到退款详情。
4. 选择“登记外部退款完成”。
5. 填写真实退款参考号、交易摘要或链上交易摘要。
6. 填写审核备注。
7. 二次确认。

禁止事项：

- 不得填写假退款单号。
- 不得伪造链上交易哈希。
- 不得在未实际退款前标记完成。
- 不得把余额支付订单登记为外部退款完成。

## 拒绝退款

1. 核对订单、支付和交付状态。
2. 选择“拒绝退款”。
3. 填写管理员备注。
4. 建议填写用户可见说明。
5. 二次确认后提交。

拒绝后状态为终态，不能随意回退。

## 标记失败

用于外部退款或余额退款处理出现真实异常。

1. 选择“标记失败”。
2. 填写失败原因摘要。
3. 不记录支付密钥、Token、完整卡密或完整支付回调。
4. 提交后保留退款记录，后续可按业务重新处理。

## 数字商品退款注意事项

- 已交付数字库存不得恢复为可售库存。
- 不删除历史交付记录。
- 未交付且仍处于 `reserved` 状态的库存可在退款成功后释放。
- 退款处理中可暂停未开始的交付。

## 审计要求

以下操作必须产生管理员审计：

- 批准余额退款
- 拒绝退款
- 取消退款
- 标记失败
- 外部人工退款登记完成
- 外部订单误走余额退款被拒绝

审计记录不得包含：

- 密码
- Token
- Service Role Key
- 完整支付回调
- 完整数字库存内容

## 手动 Migration

执行退款补强前，需要在 Supabase SQL Editor 手动执行：

```text
supabase/migrations/20260703_refund_flow_hardening.sql
```

执行后再进行真实订单退款验收。

## 验收清单

- 余额支付订单全额退款后，用户余额增加且有流水。
- 余额支付订单部分退款后，订单为部分退款。
- 重复批准同一退款不会重复加余额。
- 外部支付订单不能通过余额退款完成。
- 外部退款完成必须填写真实参考号。
- 已交付数字库存不会恢复 available。
- 用户只能查看自己的退款。
- 普通用户不能访问后台退款接口。
