# 邮件失败与重试手册

## 排查顺序

1. 检查 `/admin/notifications/email-deliveries` 的状态、错误代码和尝试次数。
2. 确认 `20260701_email_notifications.sql` 已人工执行。
3. 确认 Provider 和 `EMAIL_WORKER_SECRET` 已在服务器端配置，但不要输出值。
4. 确认当前发布模板存在且变量完整。
5. 检查收件人引用是否仍对应同一 Auth 邮箱；邮箱变化后应创建新任务。

## 错误分类

- 不重试：Provider 未配置、模板缺失、模板变量错误、地址错误、收件人引用失效、权限失败。
- 可重试：网络超时、Provider 限流、临时不可用、Provider 5xx。
- 人工检查：超过最大次数、锁长期未释放、Provider 接收但状态无法确认。

## 人工操作

- 只可重试 `pending / retrying / failed` 且未超过最大次数的任务。
- `sent` 不允许重发。
- 取消只作用于尚未发送且未在处理中的任务。
- 每次重试和取消都写入管理员审计日志。
- 不通过数据库直接把任务改成 `sent`。

## 锁恢复

Worker 领取任务后进入 `processing`。15 分钟内其他 Worker 不重复处理；超时任务可由人工重试恢复。排查时不得输出模板正文、真实邮箱、Token、密码或交付内容。

## 数据保留

建议发送尝试明细保留 90 天，任务摘要按财务和审计要求保留；清理日志不得级联删除订单、支付、退款或交付记录。清理必须先备份并人工执行。
