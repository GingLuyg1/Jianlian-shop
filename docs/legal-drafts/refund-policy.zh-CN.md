# Jianlian 退款政策

版本：`2026.0719-01`

本政策适用于通过 Jianlian（以下简称“本网站”或“本平台”）购买的数字商品及相关服务。数字商品可能具有即时查看、复制、激活、兑换、充值、使用或消耗的特点，因此退款需要结合订单的实际状态和交付情况审核。

本平台的数字商品和服务具有即时处理、自动采购或自动交付属性。订单进入相关流程后通常无法中断、撤销或退款，用户应在提交订单前再次确认全部信息。

## 一、退款申请原则

1. 提交退款申请不代表自动批准，也不代表平台已经确认商品存在问题。
2. 用户应通过本人账户提交申请，并提供订单号、退款原因、申请金额和必要证据。
3. 平台会核对订单归属、支付状态、可退金额、交付记录、商品说明及是否存在其他处理中退款。
4. 是否退款以及退款金额，以人工审核结果和适用法律为准。平台不承诺所有退款申请获得批准。

## 二、可以申请人工审核的情形

以下情况可以提交退款申请，由平台人工审核：

1. 订单已确认付款，但数字商品未交付；
2. 交付失败，且在合理核查后仍无法完成；
3. 实际交付的商品、规格或数量与订单描述明显不符；
4. 交付的数字内容在首次合理验证时即无法使用，且问题不是用户信息、操作、设备、地区或第三方账户状态导致；
5. 经核实发生重复付款；
6. 适用法律规定应当退款的其他情形。

用户应尽快报告问题。平台可以要求补充必要的错误截图、第三方平台提示或其他核验材料，但不会索取密码、私钥、助记词或完整支付密钥。

## 三、通常不支持无理由退款的情形

在不影响用户依法享有的权利前提下，下列情况通常不支持无理由退款，或需要根据实际处理状态特别审核：

1. 数字商品已经查看、复制、激活、兑换、充值、使用或消耗；
2. 商品已经按订单信息正常交付，用户因改变主意、买错商品、规格、数量或地区而申请退款；
3. 用户提交错误链接、账号、地址、地区或接收信息造成错误处理、无法撤回或交付失败；
4. 用户违反第三方平台规则、共享账号、异常使用或其他自身行为导致权益受限、失效或封禁；
5. 用户通过错误链、错误币种、错误代币合约或错误地址转账；
6. 用户伪造材料、滥用退款、已经获得重复补偿或将商品用于违法违规用途；
7. 商品页面已经清楚披露且不属于商品故障的兼容性、地区、时效或使用限制。

## 四、自动处理订单

1. 除非因本平台服务器故障、供应异常或其他原因导致订单无法完成，订单一经进入自动处理、采购、充值、开通或交付流程，原则上不支持取消或退款；是否符合退款条件仍以实际订单状态、交付证据和人工审核结果为准。
2. 用户提交错误链接、账户、地址或接收信息后，应立即携带订单号联系客服。平台会根据订单是否尚未处理、正在处理或已经完成判断能否停止或补救。
3. 同一链接、账户或目标不要同时重复下单。重复处理造成的结果需要根据每笔订单的实际状态分别核查。
4. 订单延迟或状态异常不等于自动符合退款条件，平台会先核查支付、供应、第三方平台和交付记录。

## 五、退款金额和方式

1. 申请金额不得超过系统记录的当前可退金额。已经完成或正在处理的退款会影响剩余可退金额。
2. 余额支付的退款经批准后，可以退回用户账户余额并生成相应记录。
3. USDT-BEP20 等链上支付的退款需要人工核验付款交易、订单状态和退款接收信息，不提供自动原路退款保证。
4. 链上退款产生的网络手续费可以从实际退款金额中扣除，最终到账金额以实际链上交易为准。
5. 错链、错币种、错合约或错地址转账不保证能够识别、追回或退款。区块链交易通常不可撤销。
6. 少付、多付或超过付款期限到账，首先进入相应支付异常核查流程，不自动视为退款。多付部分经人工审核确认后，可能按订单支付记录处理为账户余额或其他经双方确认的方式。
