# Jianlian Admin 商品与分类管理验收

## 验收范围

- `/admin/products`
- `/admin/categories`
- 商品新增与编辑弹窗
- 分类新增、编辑、状态切换与删除保护
- 分类管理三栏联动：一级分类、二级分类、当前分类商品
- 商品保存 API 与分类保存 API

本次未启动 `npm run dev`，未自动执行 Supabase SQL，未修改前台商城、支付、订单、余额、推广、自动发货和数字库存核心逻辑。

## 商品新增结果

- 商品写操作已统一走 `/api/admin/catalog/products` 服务端接口。
- 服务端校验管理员权限，校验商品名称、标识、分类、售价、原价、库存、状态、交付方式和排序。
- 新增成功后返回真实数据库记录，前端使用返回值刷新列表。
- 保存失败时返回中文错误，表单输入保留。
- 连续点击保存时按钮处于保存中禁用状态。
- 图片加载失败使用占位图，不阻断保存。

## 商品编辑结果

- 商品编辑走 `/api/admin/catalog/products/[productId]`。
- 编辑保存后重新读取返回的商品记录，dirty 状态归零。
- 无修改时不发送更新请求，避免“保存成功后仍提示未保存”。
- 详细说明字段不再被保存为 `null`，会保留并更新 `products.description`。
- 商品移动分类后，分类管理右侧商品栏会从旧分类列表移除并在目标分类重新读取后出现。

## 保存状态结果

- 商品表单与分类表单均保存初始快照，用快照判断 dirty 状态。
- 关闭弹窗、遮罩关闭和保存中禁用行为一致。
- 保存失败时保留 dirty 状态和输入内容。
- 保存成功后清除错误提示和弹窗临时状态。

## 一级分类结果

- 一级分类新增、编辑、启用/停用走服务端接口。
- 分类名称、标识和排序在前端校验，服务端再次校验。
- 删除一级分类前，服务端检查是否存在二级分类和直接关联商品。
- 有二级分类或商品时拒绝删除，并返回中文原因。
- 操作写入管理员审计日志。

## 二级分类结果

- 二级分类新增、编辑、移动到其他一级分类走服务端接口。
- 服务端禁止无效父级和选择自身作为父级。
- 二级分类删除前服务端检查关联商品。
- 保存后左栏、中栏和右栏会重新加载，避免重复记录。
- 操作写入管理员审计日志。

## 三栏联动结果

- `/admin/categories` 已改为三栏布局：
  - 左栏：一级分类
  - 中栏：当前一级分类下的二级分类
  - 右栏：当前一级或二级分类下的商品
- 切换一级分类会清空旧二级分类选中状态。
- 当前一级分类没有二级分类时，右栏显示直接关联该一级分类的商品。
- URL 支持 `root` 和 `category` 参数恢复有效选中状态。
- 三栏均使用内部滚动，页面主体不依赖 body 双滚动。

## 右侧商品栏结果

- 显示商品图片、名称、价格、库存、状态、更新时间和操作。
- 支持商品搜索和状态筛选。
- 支持上架、下架、售罄和编辑。
- 点击“新增商品”会带入当前一级/二级分类。
- 商品名称过长单行截断。
- 图片加载失败显示默认占位图。
- 写操作复用 `lib/supabase/admin-catalog.ts` 的统一服务入口。

## 分类删除保护结果

- 删除分类不依赖前端假判断。
- `/api/admin/catalog/categories/[categoryId]` 服务端删除前检查：
  - 子分类数量
  - 关联商品数量
- 存在依赖时返回中文错误，不删除数据。

## 发现的问题

- 商品编辑表单缺少详细说明字段，保存会把 `description` 写成 `null`。
- 商品编辑 dirty 状态曾经会把历史商品误判为有修改。
- 商品保存必须选择二级分类，导致没有二级分类的一级分类无法直接绑定商品。
- 分类管理页仍是两栏，缺少右侧商品栏。
- 商品/分类写操作直接从浏览器写 Supabase，缺少服务端删除保护和审计日志。

## 已修复的问题

- 补齐商品详细说明字段保存。
- 修复商品 dirty 状态和无修改保存逻辑。
- 支持一级分类无二级分类时直接绑定商品。
- `/admin/categories` 改为三栏联动布局。
- 新增受控后台 catalog API，统一商品/分类写入、删除保护和审计日志。
- 统一中文错误，不向浏览器暴露 Supabase 原始英文错误。

## 仍存在的问题

- 未做真实浏览器点击验收，因为本任务明确禁止启动 `npm run dev`。
- 未自动执行 Supabase SQL，也未对线上数据做新增/编辑/删除实测。
- 商品“图片列表”当前项目表结构和现有 UI 主要使用 `image_url` 主图；本次未新增字段或 migration。
