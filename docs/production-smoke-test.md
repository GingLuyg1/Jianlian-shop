# Jianlian Shop 生产冒烟测试清单

部署后必须人工执行本清单。重点确认线上页面、接口和版本 SHA 与部署版本一致。

## 版本与健康

- 打开 `https://www.jianlian.shop/api/health`。
- 确认 `version` 等于部署 commit 的短 SHA。
- 管理员访问 `/api/admin/system/version`，确认完整 SHA、分支、环境和构建时间。
- 确认主要接口无 500。
- 浏览器控制台无 `Hydration error`、`ChunkLoadError`、`MODULE_NOT_FOUND`。

## 前台页面

- 首页可打开。
- 一级类目可打开。
- 二级类目可打开。
- 三级商品列表可打开。
- 商品搜索可用。
- 商品图片显示正常，失效图有占位。
- 商品长图或详情图不撑破页面。

## 商品详情与 checkout

- 从三级商品进入商品详情。
- 复制商品详情链接重新打开。
- 刷新商品详情页。
- 有效商品不显示“商品不存在”。
- checkout 保持左侧商品详情、右侧下单卡片布局。
- 单规格商品显示价格、库存、交付方式。
- 多 SKU 商品可选择规格。
- SKU 切换后价格和库存更新。
- 售罄商品可查看但不可购买。
- 下架商品显示明确状态。

## 支付方式

- 支付方式下拉存在。
- 默认显示余额支付。
- 支付宝、微信支付、币安支付、USDT-TRC20、USDT-BEP20 显示“暂未开放”或正确配置状态。
- 未开放支付方式不能生成假二维码、假地址或假成功。
- 协议未勾选时不能提交。

## 订单链路

- 登录用户可以从商品详情点击立即购买。
- 单规格商品创建订单正常。
- 多 SKU 商品创建订单正常。
- 订单号显示正确。
- 订单详情刷新后仍存在。
- 用户不能查看其他用户订单。

## 后台

- 后台登录正常。
- 后台商品列表可打开。
- 后台商品编辑可保存。
- 保存后弹窗自动关闭。
- 列表立即显示新值。
- 刷新后新值仍存在。
- 后台订单列表可打开。
- 后台系统版本页可打开。

## 静态资源

- `/_next/static` 静态资源返回 200。
- CSS 正常加载。
- 页面不是裸 HTML。
- 移动端不出现横向滚动。

## 失败处理

如任意关键项失败：

1. 保存浏览器控制台错误。
2. 保存 PM2 日志。
3. 保存 `/api/health` 输出。
4. 记录当前线上 SHA。
5. 暂停继续部署，按 `docs/production-rollback-runbook.md` 评估回滚。
