# 充值 Provider 回调安全说明

## 当前边界

本阶段不接入新的真实 Provider，不模拟支付成功，不生成假交易号。充值自动入账必须来自真实 Provider 验签通过后的回调或管理员人工审核后的受控入账。

## 回调必须校验

真实 Provider 接入时，回调处理必须校验：

- 签名
- 商户订单号或充值单号
- Provider 交易号
- 金额
- 币种
- 支付渠道
- USDT 网络（TRC20 必须是 TRON，BEP20 必须是 BSC）
- 支付状态

验签失败、金额不一致、币种不一致、网络不一致或找不到业务单时，不得调用入账 RPC。

## 幂等要求

同一充值单只能入账一次。同一 Provider 交易号不能绑定多个充值单。重复回调应返回 Provider 期望的安全响应，但不能重复增加余额，也不能重复创建余额流水。

## 安全日志

回调日志只能保存脱敏摘要：

- 渠道
- 充值单号摘要或业务单号
- Provider 交易号
- 校验结果
- 错误代码
- 接收时间

禁止保存完整签名、密钥、Token、完整请求体中的敏感字段或用户不必要身份信息。

## 入账服务

真实回调验签通过后只能调用统一入账服务或 RPC：

```text
complete_account_recharge
```

不得在回调路由里直接更新用户余额。

## 未配置 Provider

Provider 未配置时，回调应安全失败或返回未配置提示，不得伪造成功结果。

## 人工审核与 Provider 回调关系

人工审核用于线下或手工确认渠道。真实 Provider 回调用于自动确认渠道。两者最终都必须走同一个余额入账幂等保护，不允许绕过余额流水和充值状态事务。
